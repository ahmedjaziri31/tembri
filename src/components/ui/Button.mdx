# Button Component

The Button component is a versatile, accessible button built with Radix UI primitives and styled with Tailwind CSS. It supports multiple variants, sizes, icons, and interactive states.

## Variants

The button component supports multiple visual variants:

- **primary**: Main call-to-action style with blue background
- **secondary**: Alternative action with subtle styling  
- **primary-outline**: Primary style with outline instead of fill
- **secondary-outline**: Secondary style with border
- **outlined**: Blue outline style for secondary actions
- **outline**: General outline style
- **ghost**: Minimal style for subtle actions
- **linked**: Text-only link style with underline on hover
- **link**: Standard link styling
- **destructive**: For dangerous actions

## Best Practices

### Hierarchy

- Use **primary** buttons for main actions (save, submit, confirm)
- Use **outlined** buttons for secondary actions (cancel, edit)
- Use **linked** buttons for navigation and subtle actions
- Limit to one primary button per section/form

### Icons

- Use icons to enhance meaning, not replace clear text
- Keep icon usage consistent across similar actions
- Position icons logically (left for input actions, right for forward navigation)

### Accessibility

- Provide descriptive text, not just icons
- Use proper contrast ratios for all variants
- Test with keyboard navigation and screen readers
- Include loading and disabled states for async actions
